## Recommended variants

Five (h_dim, num_layers) combos that all land inside the +/-5% band
(706k - 780k) and span the depth axis from a wide-shallow extreme
through the baseline to a narrow-deep extreme:

| role | h_dim | L | params | delta vs 743k |
| --- | ---: | ---: | ---: | ---: |
| wide / shallow (1 layer)   | 188 | 1 | 735,977 |  -0.95% |
| baseline (paper-faithful)  | 128 | 2 | 742,801 |  -0.03% |
| 3 layers, moderate width   | 104 | 3 | 754,977 |  +1.61% |
| 4 layers, narrower         |  88 | 4 | 730,369 |  -1.70% |
| narrow / deep (6 layers)   |  72 | 6 | 742,753 |  -0.03% |

All five have BiLSTM cells (the constructor is unconditionally
bidirectional), so the "L" column is the number of stacked BiLSTM
blocks looped inside `RSNN_LSTM_Reg.forward` -- not PyTorch's internal
`num_layers` arg (which is always 1 per block).

## Per-component breakdown (baseline h_dim=128, L=2)

| component | params | share |
| --- | ---: | ---: |
| BiLSTM stack (`rnn_layers`)         | 675,840 | 90.99% |
| Readout MLP (`readout`)             |  66,049 |  8.89% |
| PE encoder (`pe_encoding`)          |     144 |  0.02% |
| Atom-type embedding (`embedding`)   |     768 |  0.10% |
| **Total**                           | **742,801** | 100.00% |

The BiLSTM stack absorbs ~91% of the budget at the baseline, with the
readout MLP at ~9% and embedding+pe encoding contributing well under 1%
combined.  This means parameter scaling is dominated by `h_dim` (LSTM
weight tensors are O(h_dim^2)) and by `num_layers` (linear in stacked
BiLSTM blocks).  As a sanity check the BiLSTM count at h=128, L=2 is

```
block 0:  4 * h_dim * (h_dim + pe_out_dim + h_dim) * 2 + 4 * h_dim * 2
        = 4 * 128 * (128 + 16 + 128) * 2 + 4 * 128 * 2
        = 278,528 + 1,024  (per direction x 2 directions for the input
                            projection, plus bias terms)
block 1:  4 * h_dim * (h_dim + 2 * h_dim) * 2 + 4 * h_dim * 2
```

(the bidirectional `nn.LSTM` doubles every input-/hidden-projection
matrix; PyTorch flattens this into one 8x h_dim x in_dim weight tensor
per layer, so the closed-form numbers are easiest to obtain by
`sum(p.numel() ...)`, which is exactly what `count_params.py` does.)
